## 🔴 高风险

### R1: 反爬虫机制升级导致数据采集失败

**风险描述**:
目标网站（特别是 NMPA、EUDAMED）可能升级反爬虫机制，导致现有采集方案失效。

**影响程度**: 🔴 高  
**发生概率**: 🟡 中 (30-40%)  
**风险值**: 12/16

**具体场景**:
- NMPA 增加验证码频率
- EUDAMED 启用更复杂的 bot 检测
- IP 封禁策略收紧
- 请求频率限制降低

**缓解措施**:
1. **多策略备份**
   - 准备 3 套不同技术方案（直接请求、无头浏览器、API 接口）
   - 每套方案独立部署，可快速切换

2. **智能降级机制**
   ```typescript
   enum CrawlStrategy {
     DIRECT = 'direct',           // 直接 HTTP 请求
     PUPPETEER = 'puppeteer',     // 无头浏览器
     PROXY_POOL = 'proxy_pool',   // 代理池轮换
     API_FIRST = 'api_first',     // 优先 API
   }
   
   class AdaptiveCrawler {
     async crawl(url: string): Promise<Data> {
       for (const strategy of this.strategyPriority) {
         try {
           return await this.executeStrategy(strategy, url);
         } catch (error) {
           this.logStrategyFailure(strategy, error);
           continue;
         }
       }
       throw new Error('All strategies failed');
     }
   }
   ```

3. **监控预警**
   - 实时监控采集成功率
   - 成功率 < 90% 触发告警
   - 自动切换备用策略

**应急预案**:
- 24 小时内完成策略切换
- 准备商业代理服务（如 ScrapingBee、Bright Data）
- 与数据工程师协作，评估人工采集可行性

### R2: AI 模型准确率不达标

**风险描述**:
字段识别模型准确率无法达到 95% 目标，影响数据质量。

**影响程度**: 🔴 高  
**发生概率**: 🟡 中 (25-35%)  
**风险值**: 10/16

**根本原因**:
- 训练数据不足或质量差
- 网站结构多样性超出预期
- 多语言支持不完善

**缓解措施**:
1. **分阶段验证**
   - W6 完成原型验证（目标 85% 准确率）
   - W7 优化迭代（目标 92% 准确率）
   - W8 最终优化（目标 95% 准确率）
   - 每阶段设置 go/no-go 决策点

2. **混合策略兜底**
   ```python
   class HybridFieldExtractor:
       def extract(self, html: str) -> Dict:
           # 1. 尝试规则引擎（准确率 100%，覆盖率 70%）
           result = self.rule_based_extract(html)
           if result.confidence > 0.9:
               return result
           
           # 2. AI 模型补充（覆盖率 95%）
           ai_result = self.ai_model_extract(html)
           
           # 3. 融合结果
           return self.merge_results(result, ai_result)
   ```

3. **快速迭代机制**
   - 建立错误样本收集系统
   - 每周模型迭代更新
   - A/B 测试验证改进效果

**应急预案**:
- 准确率 < 90%：增加人工校验环节
- 准确率 < 85%：回退到纯规则引擎方案
- 启动备用模型（如 GPT-4 API 辅助标注）

### R3: 动态网站结构变更

**风险描述**:
目标网站改版或结构调整，导致已配置的解析规则失效。

**影响程度**: 🔴 高  
**发生概率**: 🟢 低 (10-20%)  
**风险值**: 8/16

**缓解措施**:
1. **结构监控**
   - 每日自动检测网站结构变化
   - 对比 DOM 树哈希值
   - 变化超过阈值触发告警

2. **自适应解析**
   ```typescript
   class AdaptiveParser {
     async parse(html: string): Promise<Data> {
       const structureHash = this.computeStructureHash(html);
       
       // 检查是否有匹配的规则
       if (this.ruleCache.has(structureHash)) {
         return this.applyRule(structureHash, html);
       }
       
       // 尝试 AI 模型自动识别
       const autoDetected = await this.aiDetectStructure(html);
       if (autoDetected.confidence > 0.8) {
         this.ruleCache.set(structureHash, autoDetected.rule);
         return this.applyRule(autoDetected.rule, html);
       }
       
       // 标记需要人工介入
       await this.alertManualIntervention(structureHash);
       throw new Error('Structure changed, manual update needed');
     }
   }
   ```

3. **版本管理**
   - 解析规则版本化
   - 支持快速回滚到历史版本
   - 记录每次结构变更

## 🟡 中风险

### R4: 数据存储性能瓶颈

**风险描述**:
大规模数据写入导致数据库性能下降。

**影响程度**: 🟡 中  
**发生概率**: 🟡 中 (30%)  
**风险值**: 6/16

**缓解措施**:
- 批量写入（每批 1000 条）
- 使用消息队列削峰填谷
- 读写分离，独立分析库
- 分区表按时间/国家分区

### R5: 并发采集导致目标网站压力

**风险描述**:
高并发采集可能被视为 DDoS 攻击，导致 IP 封禁。

**影响程度**: 🟡 中  
**发生概率**: 🟡 中 (25%)  
**风险值**: 6/16

**缓解措施**:
- 限制单网站并发数（最大 3）
- 随机延迟（1-5 秒）
- 分布式采集，分散压力
- 遵守 robots.txt 规则

---

- 针对日语单独训练子模型
- 建立多语言字段映射表
- 优先处理英语/中文内容
