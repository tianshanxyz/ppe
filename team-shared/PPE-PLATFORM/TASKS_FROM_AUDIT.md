## 任务详情

### T-001: 修复数据库迁移文件格式错误

**描述**: 数据库迁移文件使用 `.ts` 后缀但内容是纯 SQL，导致 TypeScript 编译器报错（1,453 个错误）

**文件位置**:
- `backend/ppe-backend/database/migrations/1713456789012-add-roles-permissions.ts`
- `backend/ppe-backend/database/migrations/1713456789013-add-collection-tasks.ts`
- `backend/ppe-backend/database/migrations/1713456789014-add-task-monitoring.ts`
- `backend/ppe-backend/database/migrations/1713456789015-add-quality-management.ts`
- `backend/ppe-backend/database/migrations/1713456789016-add-companies.ts`
- `backend/ppe-backend/database/migrations/1713456789017-add-regulations.ts`

**修复方案（2选1）**:
1. **方案A**: 将所有迁移文件后缀从 `.ts` 改为 `.sql`，更新 TypeORM 配置
2. **方案B**: 使用 TypeORM 的迁移 API 重写为 TypeScript 代码

**验收标准**: 运行 `npx tsc --noEmit` 不再报错

---

### T-002: 安全存储 Refresh Token（哈希）

**描述**: Refresh token 目前以明文形式存储在数据库中，如果数据库泄露，攻击者可以滥用这些 token

**文件位置**: `backend/ppe-backend/src/users/user.entity.ts:38-42`

**修复方案**:
- 添加一个方法对 refresh token 进行哈希（使用 bcrypt）
- 在存储前对 token 进行哈希
- 在验证时重新哈希并比较（与密码验证类似）

**验收标准**: 数据库中不再存储明文的 refresh token，refresh token 功能正常

---

### T-003: 加强生产环境 CORS 配置

**描述**: 当前 CORS 配置默认允许所有来源（`*`），生产环境存在安全风险

**文件位置**: `backend/ppe-backend/src/main.ts:49`

**修复方案**:
- 默认禁用 `*` 来源，要求明确配置
- 根据 NODE_ENV 进行不同配置
  - 开发环境: 可以允许 localhost
  - 生产环境: 必须配置具体的域名白名单

**验收标准**: 生产环境只允许配置的域名访问
