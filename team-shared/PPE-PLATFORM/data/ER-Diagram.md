## 四、关键索引设计

### 4.1 查询优化索引

```sql
-- 产品表索引
- idx_ppe_products_category: 按产品分类查询
- idx_ppe_products_manufacturer_id: 按制造商查询
- idx_ppe_products_data_source: 按数据来源查询
- idx_ppe_products_name_trgm: 产品名称全文搜索

-- 认证表索引
- idx_ppe_certifications_expiry_date: 认证到期查询
- idx_ppe_certifications_warning: 预警查询

-- 法规表索引
- idx_ppe_regulations_jurisdiction: 按地区查询
- idx_ppe_regulations_is_current: 当前有效法规查询
```

### 4.2 性能优化策略

1. **分区策略**: 按时间分区存储历史数据
2. **归档策略**: 定期归档过期认证记录
3. **缓存策略**: 热点数据使用 Redis 缓存
4. **读写分离**: 查询走只读副本

## 六、数据质量监控

### 6.1 质量维度

| 维度 | 指标 | 目标值 |
|------|------|--------|
| 完整性 | 必填字段填充率 | > 95% |
| 准确性 | 数据验证通过率 | > 98% |
| 一致性 | 跨表关联一致性 | > 99% |
| 时效性 | 数据更新延迟 | < 24h |
| 唯一性 | 重复记录率 | < 1% |

### 6.2 质量评分算法

```
数据质量总分 = 完整性得分 × 0.3 + 准确性得分 × 0.3 + 
              一致性得分 × 0.2 + 时效性得分 × 0.1 + 
              唯一性得分 × 0.1
```

## 七、安全设计

### 7.1 行级安全 (RLS)

- 所有核心表启用 RLS
- 公开读取策略：允许所有用户查询
- 写入控制策略：仅服务角色可写入

### 7.2 数据加密

- API 密钥加密存储
- 敏感字段加密（如 email, phone）
- 传输层 SSL/TLS 加密
